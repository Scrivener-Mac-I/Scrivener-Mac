## Overview

Scrivener's approach to long-form writing reflects an understanding that the writing process is non-linear — research informs draft text, structural changes happen at any stage, revision works on specific sections rather than reading sequentially, and the final document may need to be delivered in multiple formats for different purposes. <a href="#">Scrivener</a> accommodates all of these working patterns through its project architecture rather than fighting them with the linear assumptions of a word processor.

The <a href="#">split editor</a> divides the editing area into two independent panels each showing any document from the project — a scene and its synopsis, two versions of the same passage for comparison, the draft and a research document side by side. This split view supports the most common writing patterns where context from one part of the project needs to be visible while writing another, without the window management overhead of arranging multiple application windows for the same purpose. <a href="#">Snapshots</a> capture the current state of any document as a named version that can be compared or restored later, providing a lightweight version control system that preserves revision history without creating duplicate files or requiring external version management tools.

The <a href="#">Compile</a> function exports the assembled manuscript with formatting applied according to configured rules — manuscript format for submission, PDF for distribution, ebook format for self-publishing, Final Draft for screenplay submission. The compile settings control every aspect of the output formatting independently of how the text is formatted within Scrivener, which means the working environment can use whatever formatting is comfortable for editing while the exported document applies the exact formatting required by the submission target. <a href="#">Templates</a> for novel, screenplay, academic paper, and other project types provide preconfigured Binder structures and compile settings that match the conventions of each format, making a new project correctly configured from the first document added. Scrivener's combination of <a href="#">structural flexibility</a>, research integration, and versatile export makes it the most capable long-form writing environment available on macOS.

## Key Features

- Hierarchical <a href="#">Binder</a> organising every project section as its own document
- <a href="#">Corkboard</a> view for visual structural planning with index cards
- Collapsible <a href="#">Outliner</a> with word count and metadata columns
- Research <a href="#">storage</a> for PDFs, images, and notes within the project
- <a href="#">Split editor</a> for simultaneous view of two project documents
- <a href="#">Snapshots</a> capturing named versions of any document for comparison
- Flexible <a href="#">Compile</a> export for manuscript, ebook, PDF, and screenplay formats
- Project <a href="#">templates</a> for novel, screenplay, and academic formats
- <a href="#">Full screen</a> distraction-free writing mode with customisable environment

## Additional Information

The Compile export system's separation of working format from output format is one of Scrivener's most practically important features for writers who submit to publishers, festivals, or academic journals with specific formatting requirements. Writing in a comfortable editor format and applying submission-correct formatting at compile time means the same project can be exported to different submission formats without reformatting the manuscript for each destination.

Split editor support for simultaneous viewing of a draft and research document reflects how writing with source material actually works. Having the research document visible alongside the section being drafted eliminates the application switching that consulting external research requires, keeping the writing workflow continuous rather than interrupted by tool switching.

Templates for different project types represent accumulated knowledge about the structural conventions of each format. A novel template with parts, chapters, and front matter already structured; a screenplay template with the correct Binder hierarchy for acts, sequences, and scenes; an academic paper template with sections for abstract, chapters, and bibliography — these starting structures are correct for the format from the first document added, avoiding the organisational decisions that slow the start of a new project.
